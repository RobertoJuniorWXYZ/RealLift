# `reallift.geo.placebo.run_placebo_tests`

The elegant `run_placebo_tests` rotational matrix incorporates what is conventionally called in Data Science *In-Space Placebo* tests and/or Exact Effect Falsification Validations (*Falsification Test*). It is the fundamental trial required by the scientific method to statistically evidence that if there were no real action from your advertising campaign on the market, you **would never find a random Lift of that aggressiveness by mere causality** inducing deviations in other neutral cities.

## Signature

```python
def run_placebo_tests(
    filepath: str,
    date_col: str,
    control_geos: list,
    treatment_start_date: str,
    observed_lift: float,
    n_placebos: int = 10,
    random_state: int = None,
    cluster_idx: int = None,
    plot: bool = True,
    verbose: bool = True,
    experiment_type: str = "synthetic_control"
) -> dict
```

## MSPE Ratio Mechanics (Post/Pre)

Instead of comparing absolute Lift alone, RealLift adopts the **MSPE Ratio (Mean Squared Prediction Error)** methodology, as proposed by Abadie et al. (2010). This approach is superior because it normalizes intervention period error by that specific geo's historical fit error (pre-test).

*Important Structural Note: The placebo function inherits the strictness of selected model (`experiment_type` parameter). If you operated a Difference-in-Differences panel (`matched_did`), placebo function ensures false simulations obey the uniform 1/N weight restriction. If master model was Synthetic Control (`synthetic_control`), placebo uses Convex-optimized weights.*

1.  **Placebo Iteration**: For each geo in control pool, model tries to create exact control simulation it would use in evaluated step, treating isolated geo as if it were the unreal target of an advertising campaign.
2.  **Ratio Calculation**: For each test (including real), we calculate:
    $$
    Ratio = \frac{MSPE_{Post}}{MSPE_{Pre}}
    $$
3.  **Critical Advantage (Volatility Protection)**: Ratio use protects us in modern data and dirty *Matched DiD* scenarios. If a past placebo location already operated under strong natural noise and trend prediction wasn't excellent (high $MSPE_{Pre}$), high post-test deviation would be minimized in division, not strictly considered so "exceptional". Conversely, given a static trend (pre-test near zero), MSPE minimum error would trigger Ratio by identifying breaks. This effectively prevents false positives by punishing noisy locations.

### Probability Density (Empirical p-value formulation)

$p_{value}$ now represents probability of finding MSPE Ratio as high as observed in original treatment geo by pure chance:

$$
P_{Empirical\_p-value} = \frac{\sum_{i=1}^{P} \mathbf{I} \big(Ratio_{placebo\_i} \ge Ratio_{observed}\big)}{P}
$$

*(Where $\mathbf{I}(\cdot)$ assumes value 1 if condition is true).*

Results of $p \le 0.10$ will light up visual success metrics (`✔ High confidence:`), corroborating that anomaly generated by your campaign is statistically rare and not explainable by natural noise in other regions.
